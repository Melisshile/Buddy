# Buddy AI 2.0 — Demo Script (Competition)

**Principle:** One unforgettable workflow. Reliability over feature count.  
**Narrative:** Career progression powered by AI — not “a coding assistant that scaffolds apps.”  
**Closing line:** *Buddy doesn’t just answer questions. It continuously helps people learn, build, and grow throughout their careers.*

**Do not pitch:** Document Agent, streaming, vision, wake word, cloud chat sync, RAG.

---

## Presenter framing (before click)

> “I’m a computer systems engineering student. I want to become an AI Engineer, but I don’t know what to learn next.”

Then open Buddy.

---

## 90-second storyboard

### Scene 1 — Welcome back (0:00–0:15)

**Show:** Dashboard

**Judges see:**
- Welcome back / greeting with name
- Yesterday you completed **Authentication**
- **42%** through AI Engineering roadmap
- Current context (skills / twin), not a blank chat

**Say:** “Buddy already knows where I am in my career journey.”

**Exact CTA:** Click **Start career path** (seeds: `I want to become an AI Engineer.`)

---

### Scene 2 — AI feels active (0:15–0:30)

**User types / seeded:**

```text
I want to become an AI Engineer.
```

**Judges see status beats (not instant wall of text):**
1. Planning…
2. Updating Career Twin…
3. Creating Roadmap…
4. Saving Progress…

**Say:** “Buddy isn’t just chatting — it’s updating my Career Twin.”

---

### Scene 3 — Learning plan (0:30–0:55)

**Buddy returns (structured):**

**Learning Plan**
- Week 1 — Firebase  
- Week 2 — OpenAI APIs  
- Week 3 — AI Agents  
- Week 4 — Deployment  

**Today’s Goal:** Build a Firebase Inventory App  

**Say:** “From one career goal, Buddy produces a plan and today’s project.”

---

### Scene 4 — Build today’s project (0:55–1:15)

**User:**

```text
Build today's project.
```

**Judges see:**
- Agent activity (Career / PM / Coding / Reviewer labels as present)
- Architecture, tasks, milestones
- Tool / twin update (progress moves)

**Say:** “Now it turns today’s goal into a build plan and saves it to my twin.”

---

### Scene 5 — Visible progress (1:15–1:30)

**Back to Dashboard**

**Judges see:**
- Current Goal → Firebase Inventory App (or twin’s active project)
- Roadmap / progress nudged (e.g. 43%)
- Next skill emphasis (e.g. AI Agents)
- Today’s tasks with completed items (Generated Project / Saved Roadmap) and Begin Implementation open

**Close with winning sentence:**

> Buddy doesn’t just answer questions. It continuously helps people learn, build, and grow throughout their careers.

---

## Exact phrase bank (do not ad-lib)

| Step | String |
|------|--------|
| Career goal | `I want to become an AI Engineer.` |
| Build today | `Build today's project.` |
| Optional memory | `Remember that I prefer TypeScript` |

Avoid opening with: `Build me a Firebase inventory app.` (sounds like a generic coding bot).

---

## Pre-flight (same as COMPETITION_READINESS)

1. OpenAI live (`AI: OpenAI`, no Coach fallback badge).
2. Typed input only on stage.
3. Rehearse Scenes 1→5 twice without failure.
4. If OpenAI fails: show Dashboard + honest “coach fallback” — never fake the provider.

---

## What this demo proves (and only this)

| Proof | Scene |
|-------|-------|
| Personalization | 1 |
| Active AI (status beats) | 2 |
| Structured planning | 3 |
| Tool use + memory | 4 |
| Visible career progress | 5 |

Technologies (OpenAI, Firebase, tools) are **supporting cast**. The star is **career progression**.
